Álgebra Booleana e seus operadores lógicos.

  • O que são as operações lógicas dentro da Álgebra Booleana?

    As operações lógicas dentro da Álgebra Booleana são regras usadas para combinar ou modificar valores lógicos, que só podem ser verdadeiro (1) ou falso (0). Elas servem para analisar condições e tomar decisões em áreas como programação, matemática e circuitos digitais. Essas operações funcionam como “operações matemáticas”, mas em vez de trabalhar com números comuns, trabalham com valores de verdade.

  • Aplicações em computação e eletrônica

    A Álgebra Booleana é a base da computação digital, sendo usada para criar circuitos lógicos, como portas AND, OR, XOR e NOT, que formam os processadores e memórias dos computadores. Além disso, ela é fundamental em programação, principalmente em estruturas de decisão, filtros, validações e algoritmos que dependem de condições lógicas. Sua capacidade de representar decisões complexas com apenas valores verdadeiro/falso torna-a indispensável em tecnologia.

Operadores Lógicos

AND (E)

O operador AND só retorna verdadeiro quando todas as entradas são verdadeiras. Se qualquer entrada for falsa, o resultado será falso.

Exemplo no cotidiano:

  • Um sistema de segurança pode usar: “Alarme toca se porta aberta E janela aberta”.
    • Porta aberta = 1, Janela aberta = 0 → resultado: 0 (não toca)
    • Porta aberta = 1, Janela aberta = 1 → resultado: 1 (toca)

    OR (OU)

    O operador OR retorna verdadeiro se pelo menos uma entrada for verdadeira. Ele só retorna falso quando todas as entradas forem falsas.

    Exemplo no cotidiano:

  • Uma lâmpada que acende se sensor A OU sensor B detectar movimento.
    • A detecta movimento = 0, B detecta = 1 → resultado: 1 (liga a lâmpada)
    • A detecta = 0, B detecta = 0 → resultado: 0 (não liga)

    NOT (NÃO)

    O operador NOT inverte o valor lógico: verdadeiro vira falso e falso vira verdadeiro.

    Exemplo no cotidiano:

  • Um sistema de iluminação: “Acender luz se não estiver de dia.”
    • Dia = 1 → NOT dia = 0 (não acende)
    • Dia = 0 → NOT dia = 1 (acende)

    NAND (NÃO E)

    O operador NAND é a negação do AND. Ele retorna verdadeiro sempre que o AND seria falso e retorna falso apenas quando todas as entradas são verdadeiras.

    Exemplo no cotidiano:

  • Um alarme que desliga se todas as portas e janelas estiverem fechadas.
    • Porta fechada = 1, Janela fechada = 1 → AND = 1 → NAND = 0 (não dispara)
    • Porta aberta = 1, Janela fechada = 1 → AND = 0 → NAND = 1 (dispara)

    NOR (NÃO OU)

    O operador NOR é a negação do OR. Ele retorna verdadeiro somente quando todas as entradas forem falsas.

    Exemplo no cotidiano:

  • Um sistema de alarme: “Alarme dispara apenas se nenhum sensor detectar movimento.”
    • Sensor A = 0, Sensor B = 0 → NOR = 1 (dispara)
    • Sensor A = 1, Sensor B = 0 → NOR = 0 (não dispara)

    XOR (OU Exclusivo)

    O operador XOR retorna verdadeiro somente se uma entrada for verdadeira e a outra falsa. Se ambas forem iguais, retorna falso.

    Exemplo no cotidiano:

  • Luz de escada com dois interruptores: a luz acende se apenas um dos interruptores estiver ligado.
    • Interruptor 1 ligado, Interruptor 2 desligado → XOR = 1 (acende)
    • Ambos ligados → XOR = 0 (não acende)

    XNOR (OU Exclusivo Negado)

    O operador XNOR é o inverso do XOR. Ele retorna verdadeiro quando as entradas são iguais e falso quando são diferentes.

    Exemplo no cotidiano:

  • Comparador de senha: “A senha digitada deve ser igual à senha correta.”
    • Digitada = correta → XNOR = 1 (válido)
    • Digitada ≠ correta → XNOR = 0 (inválido)
